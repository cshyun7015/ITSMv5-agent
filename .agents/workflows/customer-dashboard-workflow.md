# 고객사 전용 대시보드 구축 워크플로우 (Customer Dashboard Workflow)

## 0. 핵심 목표
* **투명성 제공:** 고객사가 요청한 서비스 및 인시던트의 실시간 처리 현황을 공유합니다.
* **직관적 가시성:** 복잡한 인프라 지표보다는 서비스 가용성과 SLA(서비스 수준 협약) 준수 여부를 시각화합니다.
* **테넌트 격리:** 철저히 해당 고객사(Tenant ID)의 데이터만 노출되어야 합니다.

## 단계 1: 서비스 메트릭 정의
* **주도 에이전트:** ITIL Expert, PM
* **작업 내용:** * 고객사가 확인해야 할 핵심 지표(KPI) 선정: 서비스 가용성(%), 현재 장애 티켓 수, 이번 달 서비스 요청 완료율, SLA 준수 현황.
    * 테넌트별 맞춤형 위젯 구성(사용자 정의 대시보드 요소) 기획.

## 단계 2: 테넌트 브랜드 UI 설계
* **주도 에이전트:** UI/UX Designer
* **작업 내용:** * 6개 참여사의 각기 다른 CI/BI(로고, 테마 컬러)를 적용할 수 있는 유연한 대시보드 레이아웃 설계.
    * 비전문가인 현업 사용자도 한눈에 상태를 파악할 수 있는 신호등 방식(정상/주의/위험)의 인디케이터 디자인.

## 단계 3: 테넌트 필터링 API 및 DB 연동
* **주도 에이전트:** Backend Developer, DBA
* **작업 내용:** * **DBA:** 테넌트 식별자(Tenant ID) 기반의 고성능 집계 쿼리 최적화.
    * **Backend:** 특정 테넌트의 세션 정보를 기반으로 데이터가 자동 필터링되는 REST API 개발. 캐싱 기술을 적용하여 실시간 대시보드 조회 성능 확보.

## 단계 4: 프론트엔드 대시보드 구현
* **주도 에이전트:** Frontend Developer
* **작업 내용:** * React/TypeScript를 활용한 반응형 위젯 구현.
    * 테넌트별 테마 변수(CSS Variables) 연동 및 실시간 상태 업데이트(Polling 또는 WebSocket).

## 단계 5: 데이터 격리 및 가시성 검증 (QA)
* **주도 에이전트:** QA Engineer
* **작업 내용:** * **격리 검증:** A사 사용자로 로그인 시 B사의 데이터가 1건이라도 노출되는지 철저히 검증.
    * **시나리오 테스트:** 인시던트 발생 시 대시보드에 즉각 반영되는지 확인.
