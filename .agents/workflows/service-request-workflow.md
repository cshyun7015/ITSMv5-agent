## 1. 개요
[cite_start]본 워크플로우는 ITIL 4 프레임워크를 기반으로 하며, 다중 테넌트(Multi-tenancy) 환경에서 고객사와 운영사(MSP)가 서비스 요청을 효율적으로 처리할 수 있도록 설계되었습니다[cite: 2, 5, 22].

## 2. 고객사 관점 (User Portal)
[cite_start]사용자는 ITIL의 복잡성을 인지하지 않고 직관적으로 서비스를 요청하고 진행 상황을 확인하는 데 집중합니다[cite: 9, 66].

1. [cite_start]**서비스 요청 생성**: 사용자는 서비스 카탈로그에서 필요한 항목을 선택합니다[cite: 2]. [cite_start]백엔드에서 제공하는 JSON 스펙에 따라 테넌트별 맞춤형 동적 입력 폼이 화면에 렌더링됩니다[cite: 67, 68].
2. [cite_start]**내부 결재**: 백엔드 워크플로우 엔진에 의해 고객사 내부의 사전 정의된 결재선에 따라 승인이 진행됩니다[cite: 8, 57].
3. [cite_start]**진행 상태 추적**: 사용자는 본인이 요청한 티켓의 상태(접수, 이행 중, 완료 등)를 실시간으로 확인하고 알림을 받습니다[cite: 2, 11, 71].
4. [cite_start]**종료 및 평가**: 서비스 이행 완료 후 결과를 확인하고 만족도 평가를 수행하여 프로세스를 종료합니다[cite: 91].

## 3. 운영사 관점 (Admin Portal / MSP Focus)
[cite_start]운영사는 6개 고객사의 요청을 통합적으로 관리하면서도, 철저한 데이터 격리를 유지하며 서비스를 이행합니다[cite: 6, 87].

1. [cite_start]**요청 접수 및 분류**: 운영사 전용 대시보드 그리드에서 여러 고객사의 요청을 통합 확인합니다[cite: 11, 70]. [cite_start]백엔드의 테넌트 식별자 로직을 통해 운영자는 허가된 데이터에만 접근합니다[cite: 52, 54].
2. [cite_start]**담당자 배정**: 요청의 전문성에 따라 적절한 운영자에게 티켓을 할당합니다[cite: 19]. [cite_start]6개 테이블 모델 기반의 DB 스키마가 운영사와 고객사 팀의 관계를 매핑합니다[cite: 45].
3. [cite_start]**이행 및 SLA 관리**: SLA(서비스 수준 협약) 타이머를 측정하며 요청을 처리합니다[cite: 58]. [cite_start]외부 모니터링 시스템(Loki, Grafana 등)과 연동하여 인프라 상황을 참고할 수 있습니다[cite: 61, 62].
4. [cite_start]**검증 및 지식화**: QA 기준에 따라 처리 결과를 검증한 후 티켓을 종료하며, 해결 사례를 지식 베이스(DB)에 등록하여 향후 활용합니다[cite: 10, 86].

## 4. 에이전트별 구현 역할 (Agent Responsibility)

* [cite_start]**ITIL Expert**: 서비스 카탈로그 표준 프로세스 및 대시보드 기획[cite: 2, 23].
* [cite_start]**UI/UX 전문가**: 복잡성을 숨긴 직관적인 여정 설계 및 다중 테넌트 디자인 시스템 구축[cite: 9, 76, 77].
* [cite_start]**백엔드 개발자**: Java/Spring Boot 기반 워크플로우 엔진 및 멀티 테넌트 데이터 격리 로직 구현[cite: 8, 52, 53, 57].
* [cite_start]**프론트엔드 개발자**: 사용자/관리자 포털의 동적 폼 및 실시간 데이터 그리드 구현[cite: 7, 65, 67, 70].
* [cite_start]**DBA**: 6개 테이블 모델을 기반으로 고객사와 운영사 간의 관계 및 데이터 무결성 설계[cite: 41, 45, 46].
* [cite_start]**품질관리 전문가**: 고객사 및 운영사 간의 권한 분리 및 E2E 프로세스 신뢰성 검증[cite: 10, 87, 91].
* [cite_start]**프로젝트 매니저**: 모든 구축 활동이 ITIL 표준에 부합하도록 전체 트랙 조율[cite: 11, 14].
* [cite_start]**인프라 아키텍트**: 고가용성 도커 환경에서의 테넌트별 자원 할당 정책 수립 및 운영[cite: 30, 32, 33].
